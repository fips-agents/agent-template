# Conversation — 2026-04-09

## Starting Point

Ideating on an agent template for the fips-agents CLI. The goal: scaffold production-ready AI agents that deploy to OpenShift and let developers focus on prompts, tools, model selection, and evals.

## Key Discussion Points

### Monorepo with Two Templates
- Two separate directories: `templates/agent-loop/` and `templates/agentic-workflow/`
- Agent loop is the priority build
- Workflow template gets designed now, built later
- Both share a BaseAgent class

### BaseAgent Owns Everything Common
- Model calling (including JSON-structured responses)
- Tool use (local + MCP, unified interface via `self.use_tool()`)
- Prompt loading from markdown files with YAML frontmatter
- MCP client (FastMCP v3) for remote tool discovery
- Registration of tools, prompts, MCP servers
- Lifecycle methods (setup, step, teardown)
- The subclass just implements the interesting ~20-30 lines

### LlamaStack is External
- Agent speaks OpenAI-compatible chat completions API
- Points at a URL — could be LlamaStack, raw vLLM, or cloud
- Guardrails, tracing, routing are LlamaStack's concern, not the agent's
- Swapping providers is a config change, not a code change

### Infrastructure Layer
- rh-ai-quickstart/ai-architecture-charts is the assumed infra
- This project builds on top, doesn't duplicate
- Helm chart bundles just the agent (Deployment, Service, ConfigMap, optional Route)
- Infra services are pre-deployed

### Prompts as Markdown with YAML Frontmatter
- One file per prompt in `prompts/` directory
- Frontmatter has name, description, model, temperature, variables
- Body is the prompt template with `{variable}` substitution
- Easy to hand-edit; noted as harder for TypeScript consumers

### Tools
- Local tools: Python files in `tools/`, one per file, auto-discovered
- MCP tools: BaseAgent connects to configured MCP servers via FastMCP v3
- Both present the same interface — agent calls `self.use_tool("name", params)`
- Agent subclass never writes tool dispatch logic

### Workflow Template (Deferred)
- Needs a workflow manager that BaseAgent can talk back and forth to
- Manager handles state passing between nodes, manages the DAG flow
- Plan it now, build it later
- Agent loop is the first thing to ship

### Standards
- AGENTS.md follows the open standard convention (like CLAUDE.md for general agents)
- Skills, rules, commands follow coding agent conventions adapted for general-purpose agents
- Evals directory scaffolded, supports external eval harnesses

### Research Findings
- No existing project combines CLI scaffolding + BaseAgent + OpenAI-compatible API + OpenShift Helm + AGENTS.md
- rh-ai-quickstart/ai-architecture-charts is the best infra layer to build on
- LlamaStack 0.3.x has native OTel support — spans are automatic
- FastMCP v3 for the MCP client
- This fills a genuine gap in the ecosystem
